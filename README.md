# Тестовое задание на стажировку Авито (Frontend)

Привет! 👋 Это моё решение тестового задания на позицию стажера. 

Несмотря на жесткие временные рамки, из-за которых макет перенесен не полностью, я сосредоточился на реализации ключевой бизнес-логики и добавил несколько нестандартных фичей. Считаю важным показать свой подход к решению задач и архитектуре, поэтому передаю проект на проверку в текущем виде.

## 🚀 Что реализовано и исправлено

* **Багфикс сервера:** Исправлена проблема на бэкенде — сервер изначально не отдавал `id` сущностей, что ломало логику на клиенте.
* **AI-Ассистент (Умные фичи):** * Генерация продающего описания для объявления с помощью нейросетей.
    * Автоматическая оценка и рекомендация рыночной стоимости товара на основе характеристик.
* **Фильтрация и сортировка:** Частично реализована логика сортировки. Флаг «Требует доработки» работает абсолютно корректно.
* **Синхронизация данных:** Все изменения успешно сохраняются на сервер (и дублируются в локальный стор).

## 🛠 Технологии
* **Frontend:** React, Vite, Mantine UI
* **AI:** OpenRouter API
* **Backend:** Node.js (предоставленный сервер)

## ⚙️ Локальный запуск

Для запуска проекта на вашем компьютере выполните следующие шаги:

**1. Клонируйте репозиторий**
\`\`\`bash
git clone https://github.com/Valdo1824/avitoTech.git
cd твое-название-репозитория
\`\`\`

**2. Настройте переменные окружения (.env)**
Проекту требуются два файла `.env` — для клиентской и серверной части.

Создайте файл `.env` в папке `server`:
\`\`\`env
PORT=ваш_порт (например, 3000)
\`\`\`

Создайте файл `.env` в папке `client`:
\`\`\`env
VITE_AI_KEY=ваш_api_ключ_от_openrouter
\`\`\`

**3. Запустите бэкенд (Сервер)**
Откройте первый терминал, перейдите в папку сервера и выполните:
\`\`\`bash
cd server
npm install
npm start
\`\`\`

**4. Запустите фронтенд (Клиент)**
Откройте второй терминал, перейдите в папку клиента и выполните:
\`\`\`bash
cd client
npm install
npm run dev
\`\`\`

Готово! Вы великолепны! 🎉
